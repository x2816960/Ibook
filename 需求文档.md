# Ibook - 待办任务管理系统 需求文档

> 版本：v1.3
> 日期：2026-03-23
> 状态：更新中

---

## 1. 项目概述

### 1.1 项目名称

Ibook - 待办任务管理系统

### 1.2 项目目标

开发一个基于 Web 的多用户待办任务管理系统，支持任务的创建、编辑、删除、状态管理、筛选排序等功能，提供直观的任务统计面板，帮助用户高效管理日常工作任务。

### 1.3 目标用户

- 个人用户：管理个人日常待办事项
- 小型团队：每个成员独立管理自己的任务

---

## 2. 系统架构要求

### 2.1 技术选型建议

| 层级 | 建议方案 | 说明 |
|------|---------|------|
| 前端 | Vue 3 + Element Plus | 组件丰富，拖拽支持好 |
| 后端 | Python Flask / FastAPI | 轻量、易部署、Ubuntu 兼容性好 |
| 数据库 | SQLite / PostgreSQL | SQLite 零配置适合单机部署，PostgreSQL 适合多用户 |
| 部署 | Docker / 直接部署 | 优先支持 Ubuntu 24.04 |

### 2.2 部署环境

- **优先适配**：Ubuntu 24.04 LTS
- **兼容适配**：Ubuntu 22.04 LTS、Ubuntu 20.04 LTS
- 浏览器支持：Chrome、Firefox、Edge 最新两个主要版本

## 3. 功能需求

### 3.1 用户管理模块

#### 3.1.1 默认管理员账户

- 系统首次部署时自动创建管理员账户
- 默认账户：`admin`，首次部署时通过安装脚本交互式设置管理员密码（密码需符合系统密码规则：8-32 位，包含字母和数字）
- 若通过 Docker 部署，支持通过环境变量 `ADMIN_PASSWORD` 设置初始密码
- 管理员权限：
  - 查看所有注册用户列表
  - 禁用/启用用户账户
  - 重置用户密码
  - 查看系统级统计数据（总用户数、总任务数等）

#### 3.1.2 用户注册

- 注册页面字段：
  - 用户名（必填，4-20 位，字母/数字/下划线）
  - 密码（必填，8-32 位，需包含字母和数字）
  - 确认密码（必填，与密码一致）
  - 邮箱（选填，用于密码找回）
  - 昵称（选填，用于页面显示）
- 注册校验：
  - 用户名唯一性检查
  - 密码强度提示
  - 验证码防刷（可选，建议简单图形验证码）
- 注册成功后自动跳转登录页

#### 3.1.3 用户登录

- 登录页面字段：
  - 用户名/邮箱（必填）
  - 密码（必填）
  - 记住我（可选，延长会话有效期）
- 登录逻辑：
  - 密码错误提示（不透露具体是用户名还是密码错误）
  - 连续 5 次登录失败，锁定账户 15 分钟（锁定期满后自动解锁，管理员可手动提前解锁）
  - 登录成功后跳转至任务主页
- 会话管理：
  - 基于 Token（JWT）的认证方式
  - 默认会话有效期 24 小时
  - 勾选"记住我"后有效期延长至 7 天
  - 退出登录时将 Token 加入服务端黑名单（基于数据库或内存缓存），黑名单记录在 Token 过期后自动清理

#### 3.1.4 个人设置

- 修改昵称
- 修改密码（需输入旧密码验证）
- 退出登录

### 3.2 任务管理模块

#### 3.2.1 任务数据模型

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | 整数 | 自动 | 任务唯一标识 |
| title | 字符串 | 是 | 任务标题，最长 100 字符 |
| description | 文本 | 否 | 任务描述，最长 2000 字符 |
| detail_content | 文本 | 否 | 详细信息，Markdown 格式，媒体文件统一通过附件表存储，Markdown 中仅保存附件引用链接 |
| priority | 枚举 | 是 | 优先级：高/中/低 |
| status | 枚举 | 是 | 状态：待办/进行中/已完成/已取消 |
| due_date | 日期时间 | 否 | 截止时间，可为空表示"无限期" |
| is_indefinite | 布尔 | 否 | 是否无限期（true 时忽略 due_date） |
| sort_order | 整数 | 自动 | 排序序号 |
| created_at | 日期时间 | 自动 | 创建时间 |
| updated_at | 日期时间 | 自动 | 更新时间 |
| user_id | 整数 | 自动 | 所属用户 |

**附件数据模型（关联表）：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | 整数 | 自动 | 附件唯一标识 |
| task_id | 整数 | 是 | 关联的任务 ID |
| file_name | 字符串 | 是 | 原始文件名 |
| file_path | 字符串 | 是 | 服务器存储路径 |
| file_size | 整数 | 自动 | 文件大小（字节） |
| file_type | 枚举 | 自动 | 类型：image / video / other |
| mime_type | 字符串 | 自动 | MIME 类型（如 image/png） |
| created_at | 日期时间 | 自动 | 上传时间 |

#### 3.2.2 任务列表展示

- 列表视图，每行显示：
  - 任务标题
  - 任务描述（截断显示，悬浮查看完整内容）
  - 优先级标签（高=红色、中=橙色、低=绿色）
  - 状态标签（待办=灰色、进行中=蓝色、已完成=绿色、已取消=灰色删除线）
  - 截止时间（过期任务红色高亮显示，无限期任务显示"无限期"标签）
  - 附件图标（有附件时显示📎图标及附件数量，点击可展开查看）
  - 操作按钮（编辑、删除、状态切换）
- 详细信息查看：
  - 点击已保存的任务可展开查看详细信息
  - 展开区域以 Markdown 渲染方式显示详细内容
  - 图片点击后弹出放大显示（支持缩放、旋转等操作）
  - 视频点击后在线播放（支持全屏、倍速播放）
  - 附件点击后提示下载
- 分页展示，每页默认 20 条，支持切换每页条数（10/20/50）
- 空状态提示："暂无任务，点击新建"

#### 3.2.3 任务新增

- 弹窗/抽屉形式的新增表单
- 表单字段：
  - 标题（必填）
  - 描述（选填，支持多行文本）
  - 优先级（必填，默认"中"）
  - 截止时间（选填，日期时间选择器）
  - 无限期选项（勾选后截止时间置空，表示任务无截止日期限制）
  - 附件上传（选填，支持上传图片、视频、文件）
- 快速新建模式：
  - 新建任务后显示"编写详细信息"按钮
  - 点击后展开详细信息编辑区域
- 详细信息编辑功能（Markdown 编辑器）：
  - 支持 Markdown 语法的富文本编辑
  - 工具栏提供常用格式按钮（标题、加粗、斜体、列表、链接、代码块等）
  - 支持插入图片：上传图片后自动插入 Markdown 图片语法，点击图片可放大显示
  - 支持插入视频：上传视频后插入播放器，点击可在线播放
  - 支持插入附件：上传附件后插入下载链接，点击提示下载
  - 提供实时预览功能，编辑时可切换编辑/预览/分屏模式
  - 保存后以 Markdown 格式存储
- 附件上传要求：
  - 支持的图片格式：JPG、PNG、GIF、BMP、WebP
  - 支持的视频格式：MP4、AVI、MOV、MKV
  - 支持的其他附件格式：PDF、DOC/DOCX、XLS/XLSX、PPT/PPTX、TXT、ZIP、RAR
  - 单个文件大小限制：图片 ≤ 10MB，视频 ≤ 200MB，其他附件 ≤ 50MB（默认值，管理员可配置）
  - 每个任务最多上传 10 个附件
  - 上传时显示进度条，支持取消上传
  - 上传完成后显示文件缩略图（图片）或文件图标（视频/其他）
  - 支持拖拽文件到上传区域
  - 支持点击选择文件上传
- 新增任务默认状态为"待办"
- 新增成功后列表自动刷新

#### 3.2.4 任务编辑

- 点击编辑按钮弹出编辑表单，回填原有数据
- 可编辑字段：标题、描述、详细信息（Markdown）、优先级、截止时间、无限期选项、附件
- 详细信息编辑：
  - 提供 Markdown 编辑器，同新增任务功能
  - 编辑器中可继续编辑已有的 Markdown 内容
  - 支持追加上传图片、视频、附件到详细信息中
  - 实时预览编辑效果
- 附件编辑：
  - 查看已上传的附件列表
  - 可追加上传新附件（不超过总数限制）
  - 可删除已有附件（确认后删除）
- 状态不在编辑表单中修改（通过独立的状态切换操作）

#### 3.2.5 任务删除

- 点击删除按钮弹出确认对话框
- 确认后执行软删除（数据保留，前端不展示）
- 已完成和已取消的任务可批量清理
- 回收站功能：
  - 提供回收站入口，可查看所有已软删除的任务
  - 支持从回收站恢复任务（恢复为"待办"状态）
  - 支持从回收站彻底删除（永久删除，不可恢复）
  - 回收站中的任务超过 30 天自动永久删除
- 附件清理策略：
  - 任务永久删除时，同步删除关联的附件文件和数据库记录
  - 软删除期间附件文件保留，不占用附件数量配额

#### 3.2.6 状态切换

- 支持的状态流转：
  ```
  待办 → 进行中
  待办 → 已取消
  进行中 → 已完成
  进行中 → 待办（回退）
  已完成 → 进行中（回退，用于误操作恢复）
  已取消 → 待办（恢复）
  ```
- 状态切换方式：
  - 点击状态标签弹出可选状态列表
  - 或通过下拉菜单选择目标状态
- 不允许的流转自动禁用（如"已完成"不能直接变为"待办"）

#### 3.2.7 任务筛选

- 筛选条件（可组合）：
  - 按状态筛选：全部 / 待办 / 进行中 / 已完成 / 已取消
  - 按优先级筛选：全部 / 高 / 中 / 低
- 筛选条件以标签页或下拉框形式展示在列表上方
- 筛选结果实时更新，无需手动刷新
- 支持关键词搜索（搜索标题和描述）

#### 3.2.8 拖拽排序

- 在任务列表中支持鼠标拖拽调整任务顺序
- 拖拽时显示拖拽占位符，视觉反馈清晰
- 松开鼠标后自动保存新排序到服务器
- 排序结果持久化，刷新页面后保持
- 筛选状态下拖拽排序仅影响当前筛选结果内的顺序

### 3.3 任务统计模块

#### 3.3.1 统计面板

在任务列表页面顶部展示统计卡片：

| 统计项 | 说明 |
|--------|------|
| 任务总数 | 当前用户的所有未删除任务数量 |
| 待办数量 | 状态为"待办"的任务数量 |
| 进行中数量 | 状态为"进行中"的任务数量 |
| 已完成数量 | 状态为"已完成"的任务数量 |
| 已取消数量 | 状态为"已取消"的任务数量 |
| 今日到期 | 截止时间为今天的未完成任务数量 |
| 已过期 | 截止时间已过但未完成的任务数量（红色警示） |

#### 3.3.2 统计展示样式

- 以卡片形式横向排列
- 各卡片显示数字和标签
- 点击卡片可快速筛选对应状态的任务（如点击"待办"卡片自动筛选待办任务）

### 3.4 管理员功能模块

#### 3.4.1 用户管理（仅管理员）

- 查看用户列表（用户名、昵称、邮箱、注册时间、状态）
- 禁用/启用用户
- 重置用户密码

#### 3.4.2 系统配置（仅管理员）

- 附件上传限制配置：
  - 图片最大文件大小（默认 10MB，可调整范围 1MB - 100MB）
  - 视频最大文件大小（默认 200MB，可调整范围 10MB - 2GB）
  - 其他附件最大文件大小（默认 50MB，可调整范围 1MB - 500MB）
  - 每个任务最大附件数量（默认 10 个，可调整范围 1 - 50）
- 配置修改后即时生效，已上传的文件不受影响
- 配置页面显示当前存储用量统计（总附件数、总占用空间）

#### 3.4.3 系统统计（仅管理员）

- 总注册用户数
- 总任务数
- 各状态任务占比

### 3.5 备份与恢复模块

#### 3.5.1 数据导出工具

- 提供数据库导出脚本/工具
- 导出内容：
  - 数据库完整数据（用户、任务、附件记录等）
  - 所有附件文件（图片、视频、其他文件）
- 导出格式：
  - 数据库导出为 SQL 文件或 SQLite 数据库文件副本
  - 附件文件保持原有目录结构打包
  - 最终输出为压缩包（.tar.gz 或 .zip）
- 导出方式：
  - 命令行脚本：`./export.sh [output_path]`
  - 管理员界面一键导出按钮
- 导出文件命名：`ibook_backup_YYYYMMDD_HHMMSS.tar.gz`

#### 3.5.2 数据导入工具

- 提供数据导入脚本/工具
- 导入功能：
  - 校验备份文件完整性
  - 恢复数据库数据
  - 恢复附件文件到指定目录
- 导入方式：
  - 命令行脚本：`./import.sh [backup_file]`
  - 管理员界面上传备份文件导入
- 安全措施：
  - 导入前确认提示（警告数据将被覆盖）
  - 可选增量导入（仅导入不存在的数据）
  - 导入完成后显示导入结果报告

#### 3.5.3 定期备份（可选）

- 支持 cron 定时任务配置
- 自动清理超过指定天数的旧备份文件

## 4. 非功能需求

### 4.1 性能要求

- 页面首次加载时间 < 3 秒
- 任务列表请求响应时间 < 500ms（1000 条任务以内）
- 拖拽排序保存响应时间 < 300ms
- 支持至少 50 个并发用户

### 4.2 安全要求

- 密码使用 bcrypt 加盐哈希存储，禁止明文存储
- API 接口使用 JWT Token 认证
- 防止 SQL 注入、XSS 攻击
- 用户只能访问自己的任务数据（接口级权限校验）
- HTTPS 支持（生产环境）
- **附件下载说明**：附件上传接口需认证，附件下载接口无需认证（简化 URL 避免 token 过期问题）

### 4.3 可用性要求

- 响应式布局，支持 PC 端浏览器（移动端适配为低优先级）
- 操作反馈及时（加载状态、成功/失败提示）
- 表单校验提示清晰

### 4.4 Ubuntu 兼容性要求

| Ubuntu 版本 | 优先级 | Python 版本 | Node.js 版本 | 备注 |
|-------------|--------|-------------|-------------|------|
| 24.04 LTS | **最高** | 3.12+ | 18+ | 主要开发和测试环境 |
| 22.04 LTS | 高 | 3.10+ | 16+ | 广泛使用 |
| 20.04 LTS | 中 | 3.8+ | 14+ | 仍在支持周期 |

- 提供一键安装脚本（`install.sh`），自动检测系统版本并安装依赖
- 提供 Docker 部署方案作为跨版本兼容的备选方案

## 5. 页面设计

### 5.1 页面清单

| 页面 | 路径 | 访问权限 | 说明 |
|------|------|---------|------|
| 登录页 | `/login` | 公开 | 用户登录 |
| 注册页 | `/register` | 公开 | 用户注册 |
| 任务主页 | `/tasks` | 登录用户 | 任务列表、统计、筛选、排序 |
| 回收站 | `/tasks/trash` | 登录用户 | 查看/恢复/永久删除已软删除的任务 |
| 个人设置 | `/settings` | 登录用户 | 修改密码、昵称 |
| 用户管理 | `/admin/users` | 管理员 | 管理用户账户 |
| 系统配置 | `/admin/config` | 管理员 | 配置附件上传大小限制等 |
| 系统统计 | `/admin/stats` | 管理员 | 查看系统级统计 |
| 备份恢复 | `/admin/backup` | 管理员 | 数据导出与导入 |

### 5.2 页面布局

```
+------------------------------------------+
|  顶部导航栏（Logo、用户信息、退出登录）     |
+------------------------------------------+
|  统计卡片区域                              |
|  [总数] [待办] [进行中] [已完成] [过期]     |
+------------------------------------------+
|  筛选区域                                  |
|  [状态筛选] [优先级筛选] [搜索框] [新增按钮] |
+------------------------------------------+
|  任务列表区域                              |
```

## 6. API 接口设计（概要）

### 6.1 认证接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 用户注册 |
| POST | `/api/auth/login` | 用户登录 |
| POST | `/api/auth/logout` | 退出登录 |
| PUT | `/api/auth/password` | 修改密码 |

### 6.2 任务接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/tasks` | 获取任务列表（支持筛选、分页） |
| POST | `/api/tasks` | 新增任务 |
| PUT | `/api/tasks/:id` | 编辑任务 |
| DELETE | `/api/tasks/:id` | 删除任务 |
| PATCH | `/api/tasks/:id/status` | 切换任务状态 |
| PUT | `/api/tasks/sort` | 保存拖拽排序 |
| GET | `/api/tasks/stats` | 获取任务统计 |

### 6.3 附件接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/tasks/:id/attachments` | 上传附件（multipart/form-data） |
| GET | `/api/tasks/:id/attachments` | 获取任务附件列表 |
| GET | `/api/attachments/:id/download` | 下载/预览附件 |
| DELETE | `/api/attachments/:id` | 删除附件 |

### 6.4 管理员接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/users` | 获取用户列表 |
| PATCH | `/api/admin/users/:id` | 禁用/启用用户 |
| POST | `/api/admin/users/:id/reset-password` | 重置用户密码 |
| GET | `/api/admin/stats` | 系统统计 |
| GET | `/api/admin/config` | 获取系统配置（附件大小限制等） |
| PUT | `/api/admin/config` | 更新系统配置 |

### 6.5 备份恢复接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/admin/backup/export` | 导出数据备份（生成下载链接） |
| GET | `/api/admin/backup/download/:filename` | 下载备份文件 |
| POST | `/api/admin/backup/import` | 导入数据备份（上传备份文件） |
| GET | `/api/admin/backup/list` | 获取备份文件列表（可选） |

## 8. 验收标准

- [ ] 管理员默认账户可正常登录，首次部署时交互式设置密码
- [ ] 新用户可注册账户并登录
- [ ] 任务可正常新增、编辑、删除
- [ ] 任务状态可按规则正确流转（包括已完成可回退到进行中）
- [ ] 筛选功能可按状态、优先级过滤任务
- [ ] 拖拽排序后刷新页面排序保持不变
- [ ] 统计面板数据准确实时更新
- [ ] 不同用户之间任务数据完全隔离
- [ ] 在 Ubuntu 24.04 上可一键部署运行
- [ ] 密码加密存储，接口认证有效
- [ ] JWT 退出登录后 Token 立即失效（黑名单机制）
- [ ] 任务详细信息支持 Markdown 编辑，可插入图片、视频、附件（媒体统一走附件表存储）
- [ ] 图片点击可放大显示，视频可在线播放，附件可下载
- [ ] Markdown 预览中图片始终可正常显示（下载 URL 不依赖 token）
- [ ] 任务可设置无限期截止时间
- [ ] 软删除的任务可在回收站中查看、恢复或永久删除
- [ ] 回收站任务超过 30 天自动永久删除，附件同步清理
- [ ] 数据导出工具可导出完整数据库及所有附件
- [ ] 数据导入工具可恢复数据库及附件
- [ ] doc 目录下包含完整的项目文档（README、API、部署、更新日志）
